# 출시 체크리스트 - 성인인증 추가 후 출시 가능 여부

## 📋 현재 상태 분석

### ✅ 이미 구현된 것
1. **전화번호 인증** (Firebase Phone Auth)
   - SMS 인증 코드 발송/검증
   - 전화번호 기반 로그인

2. **나이 검증**
   - 만 19세 이상만 가입 가능
   - 나이 입력 및 검증 로직

3. **이용약관/개인정보 처리방침**
   - 성인인증 관련 조항 포함
   - 미성년자 차단 명시

4. **콘텐츠 필터링**
   - 부적절한 내용 필터링
   - 신고 기능

### ❌ 부족한 것 (출시 전 필수)
1. **실제 본인인증 API 연동**
   - 현재: 나이 입력만 받음 (사용자가 거짓 입력 가능)
   - 필요: 나이스 본인인증, 아이핀 등 실제 본인인증

2. **앱스토어/플레이스토어 연령 제한 설정**
   - iOS: 17+ 또는 18+ 연령 제한 설정 필요
   - Android: 성인 콘텐츠 분류 필요

3. **법적 요구사항**
   - 정보통신망법 준수
   - 청소년 보호법 준수
   - 개인정보 보호법 준수

---

## 🚨 출시 가능 여부 판단

### ⚠️ 현재 상태로는 출시 불가능

**이유:**
1. **실제 본인인증 없음**
   - 나이 입력만으로는 미성년자가 가입 가능
   - 법적 책임 문제 발생 가능
   - 앱스토어 심사에서 거부될 가능성 높음

2. **법적 요구사항 미충족**
   - 성인 전용 서비스는 반드시 본인인증 필요
   - 단순 나이 입력은 법적 효력 없음

---

## ✅ 출시를 위해 필요한 작업

### 1. 본인인증 API 연동 (필수)

#### 옵션 A: 나이스 본인인증 (추천)
- **장점**: 가장 널리 사용, 신뢰도 높음
- **비용**: 유료 (건당 약 300-500원)
- **구현**: 나이스페이먼츠 API 연동
- **필요한 것**: 사업자등록증, API 키 발급

#### 옵션 B: 아이핀 (i-PIN)
- **장점**: 무료, 간편
- **단점**: 신뢰도 상대적으로 낮음
- **구현**: 한국정보인증 API 연동

#### 옵션 C: 통신사 본인인증
- **장점**: 신뢰도 매우 높음
- **단점**: 비용 높음, 구현 복잡
- **구현**: 각 통신사 API 연동

### 2. 앱스토어 설정 (필수)

#### iOS App Store
```json
// app.json에 추가 필요
"ios": {
  "infoPlist": {
    "ITSAppUsesNonExemptEncryption": false,
    // 연령 제한 설정
  }
}
```
- App Store Connect에서 연령 제한 설정 (17+ 또는 18+)
- 성인 콘텐츠 분류
- 심사 시 성인인증 절차 설명 필요

#### Google Play Store
- 콘텐츠 등급: 성인용 (18+)
- 개인정보 처리방침 URL 필수
- 성인인증 절차 설명 필요

### 3. 개인정보 처리방침 업데이트 (필수)
- 본인인증 수집 정보 명시
- 본인인증 제공업체 명시
- 개인정보 보관 기간 명시

### 4. 추가 보안 강화 (권장)
- 본인인증 결과 암호화 저장
- 재인증 정기 요구 (예: 1년마다)
- 의심스러운 계정 추가 검증
