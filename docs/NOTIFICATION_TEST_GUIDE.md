# 알림 테스트 가이드

알림 기능을 테스트하는 방법을 안내합니다.

## 📱 테스트 화면 사용

### 1. 테스트 화면 접근
1. 앱 실행 후 "더보기" 탭으로 이동
2. "알림 테스트" 메뉴 클릭

### 2. 테스트 항목

#### 알림 권한 확인
- **목적**: 알림 권한이 제대로 허용되었는지 확인
- **방법**: "알림 권한 확인" 버튼 클릭
- **예상 결과**: "알림 권한이 허용되었습니다." 메시지 표시

#### 푸시 토큰 가져오기
- **목적**: Expo Push 토큰을 가져와서 Firestore에 저장
- **방법**: "푸시 토큰 가져오기" 버튼 클릭
- **예상 결과**: 
  - 토큰이 표시됨 (예: `ExponentPushToken[xxxxx...]`)
  - Firestore의 `users/{userId}` 문서에 `expoPushToken` 필드가 저장됨
- **확인 방법**: Firebase Console에서 사용자 문서 확인

#### 메시지 알림 테스트
- **목적**: 메시지 알림이 제대로 작동하는지 확인
- **방법**:
  1. 발신자 이름과 메시지 내용 입력 (기본값 사용 가능)
  2. "메시지 알림 전송" 버튼 클릭
  3. **앱을 백그라운드로 보냄** (홈 버튼 누르기)
  4. 알림 확인
- **예상 결과**: 
  - 알림이 표시됨
  - 알림을 탭하면 채팅방으로 이동

#### 좋아요 알림 테스트
- **목적**: 좋아요 알림이 제대로 작동하는지 확인
- **방법**:
  1. "좋아요 알림 전송" 버튼 클릭
  2. **앱을 백그라운드로 보냄** (홈 버튼 누르기)
  3. 알림 확인
- **예상 결과**: 
  - 알림이 표시됨
  - 알림을 탭하면 프로필 화면으로 이동

#### 알림 배지 초기화
- **목적**: 앱 아이콘의 알림 배지(뱃지) 제거
- **방법**: "알림 배지 초기화" 버튼 클릭
- **예상 결과**: 앱 아이콘의 빨간 배지 숫자가 0으로 변경됨

## 🔍 실제 시나리오 테스트

### 시나리오 1: 메시지 수신 알림 (백그라운드)

**준비**:
1. 두 개의 기기 또는 시뮬레이터 준비
2. 각각 다른 계정으로 로그인

**테스트 단계**:
1. 기기 A: 앱을 백그라운드로 보냄 (홈 버튼)
2. 기기 B: 기기 A 사용자와 채팅 시작
3. 기기 B: 메시지 전송
4. 기기 A: 알림 확인

**예상 결과**:
- 기기 A에 알림이 표시됨
- 알림을 탭하면 채팅방으로 이동

### 시나리오 2: 좋아요 알림 (백그라운드)

**준비**:
1. 두 개의 기기 또는 시뮬레이터 준비
2. 각각 다른 계정으로 로그인

**테스트 단계**:
1. 기기 A: 앱을 백그라운드로 보냄
2. 기기 B: 기기 A 사용자 프로필에서 좋아요 누르기
3. 기기 A: 알림 확인

**예상 결과**:
- 기기 A에 좋아요 알림이 표시됨
- 알림을 탭하면 좋아요를 누른 사용자 프로필로 이동

### 시나리오 3: 포그라운드 알림 (알림 표시 안 됨)

**테스트 단계**:
1. 앱이 포그라운드에 있는 상태에서 메시지 수신
2. 알림 확인

**예상 결과**:
- 알림이 표시되지 않음 (정상 동작)
- 채팅방에서 메시지가 바로 표시됨

### 시나리오 4: 현재 채팅방에서 알림 방지

**테스트 단계**:
1. 특정 채팅방 열기
2. 앱을 백그라운드로 보냄
3. 같은 채팅방에서 메시지 수신

**예상 결과**:
- 알림이 표시되지 않음 (정상 동작)
- 현재 열려있는 채팅방에서는 알림이 표시되지 않아야 함

## 🐛 문제 해결

### 알림이 오지 않는 경우

1. **알림 권한 확인**
   - iOS: 설정 > 알림 > 앱 이름
   - Android: 설정 > 앱 > 알림

2. **앱 재빌드**
   ```bash
   npx expo run:ios
   # 또는
   npx expo run:android
   ```
   - `expo-notifications`는 네이티브 모듈이므로 재빌드 필요

3. **Firestore 권한 확인**
   - Firebase Console에서 Firestore 규칙 확인
   - `expoPushToken` 필드 저장 권한 확인

4. **로그 확인**
   - 개발자 도구에서 콘솔 로그 확인
   - "푸시 토큰 저장 완료" 메시지 확인

### 알림을 탭했을 때 화면 이동이 안 되는 경우

1. **알림 리스너 확인**
   - `AppNavigator.tsx`의 알림 리스너 설정 확인
   - 로그인 상태인지 확인

2. **네비게이션 확인**
   - 알림 데이터의 `chatRoomId` 또는 `likerId` 확인
   - 해당 데이터가 존재하는지 확인

### 백그라운드에서 알림이 오지 않는 경우

**현재 제한사항**:
- 앱이 **완전히 종료된 상태**에서는 알림이 오지 않습니다
- 이는 Cloud Functions를 설정해야 해결됩니다

**임시 해결책**:
- 앱을 백그라운드로만 보내서 테스트 (완전 종료하지 않음)

2. 푸시 토큰 입력 (Firestore에서 복사)

## 📝 참고사항

- **앱 완전 종료 시 알림**: Cloud Functions 설정 필요 (별도 문서 참조)
- **실제 기기 테스트 권장**: 시뮬레이터에서는 일부 기능이 제한될 수 있음
- **배터리 최적화**: Android에서 배터리 최적화가 켜져 있으면 백그라운드 알림이 제한될 수 있음
