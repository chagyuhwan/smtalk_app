# 배포 전 추가 기능 제안

배포 전에 추가하면 좋을 기능들을 우선순위별로 정리했습니다.

---

## 🔴 필수 기능 (배포 전 반드시 필요)

### 1. 알림 설정 화면 구현
**현재 상태**: UI만 있고 기능이 미구현

**필요한 기능**:
- [ ] 알림 ON/OFF 토글
- [ ] 알림 종류별 설정
  - 새 메시지 알림
  - 좋아요 알림
  - 신고 처리 완료 알림
- [ ] 알림 설정 Firestore 저장
- [ ] 알림 설정에 따른 실제 알림 제어

**구현 위치**: `screens/MoreScreen.tsx`의 "알림 설정" 메뉴 연결

**예상 작업 시간**: 2-3시간

---

### 2. 포인트 사용 내역 조회
**현재 상태**: 충전 내역만 있고 사용 내역 없음

**필요한 기능**:
- [ ] 포인트 사용 내역 화면 생성
  - 채팅 시작 (50P 차감)
  - 출석체크 (50P 지급)
  - 포인트 충전
- [ ] 사용 내역 Firestore 저장
- [ ] 내역 조회 화면 (날짜별, 유형별 필터링)
- [ ] "더보기" 화면에 "포인트 내역" 메뉴 추가

**구현 위치**: 
- 새 화면: `screens/PointHistoryScreen.tsx`
- Firestore에 사용 내역 저장 로직 추가

**예상 작업 시간**: 3-4시간

---

### 3. 푸시 알림 완전 구현
**현재 상태**: NotificationService는 있지만 실제 메시지 알림이 완전히 연결되지 않음

**필요한 기능**:
- [ ] 새 메시지 수신 시 알림 표시
- [ ] 좋아요 받았을 때 알림
- [ ] 백그라운드 알림 처리
- [ ] 알림 탭 시 해당 화면으로 이동
- [ ] 현재 채팅방에서는 알림 표시 안 함 (이미 구현됨)

**구현 위치**: `context/ChatContext.tsx`의 메시지 수신 로직

**예상 작업 시간**: 2-3시간

## 💡 빠른 구현 팁

### 알림 설정 화면
- AsyncStorage 또는 Firestore에 설정 저장
- NotificationService와 연동하여 실제 알림 제어

### 포인트 사용 내역
- Firestore에 `pointTransactions` 컬렉션 생성
- 채팅 시작, 출석체크 등에서 트랜잭션 기록
- 리스트 화면으로 조회

### 푸시 알림
- ChatContext의 메시지 수신 로직에 알림 추가
- 현재 채팅방 ID와 비교하여 알림 표시 여부 결정
